Database

The database contains data of 200 DNA sequences provided by BLAST. In total 174 sequences had one or more results. Only the first 30 results per sequence have been saved to the database. If a sequence had less than 30 results, all of its results have been saved. The database can be searched with multiple parameters.

Search: search the database with multiple paramters. A choice can be made aided by checkboxes what to search in the database, e.g. header, defenition etc.
Count: search the database, and count how many times an item occurs. There can only be chosen 1 checkbox.
Organism: an option to choose to search in the organism database. The choices are: species, genus and family. (See 'what to look for', for more information)
Protein: an option to choose to search in the organism database. The choices are: accession code and defenition. (See what to look for, for more information)
What to look for: an option to filter something in the database
If the previous choice was organism: the options to search for are a species (e.g. Gammaproteobacteria bacterium), a genus (e.g. Gammaproteobacteria) or a family (e.g. Proteobacteria)
If the previous choise was protein: the options to search for are either a defenition (e.g. outer membrane protein assembly factor BamB) or an accession code (e.g. TDJ28483)
Limit: an option to limit the amount of results that will be shown in a table

Search: search the database with all the selected parameters The options for what to show in the table:

  • Header: The header of each DNA sequence
  • Defenition: The defenition of the protein
  • Orgaism_species: The name of the species
  • Ident_perc: The identity percentage, calculated by the amout of identities divided by the length of the alignment
  • Accession: The accession code (NCBI) of the protein
  • Sequence: The DNA sequence that is used to BLAST
  • Organism_genus: The name of the genus, which is
  • Organism_family: The name of the family, which is
  • Query_cov: The query coverage, calculated by (end of hit minus begin of hit) divided by the length of the DNA sequence
  • E_value: The e-value which stands for the amout of random sequences in the NCBI database with the same of a higher bitscore

About

How does it all work and how has it been made?

Morgan

First of all, a Python script was made to automatically BLAST 200 DNA sequences, in which the tool biopyton was used. All of the data which BLAST provided was saved to an XML file. Then, a Python function was made to parse the XML file, and retrieve the important data. In addition, a Python function was made to automatically insert the retrieved data into a self-created database.
Secondly, this website was created, aided by Python and the Flask package.
