Database

The database contains data of 200 DNA sequences provided by BLAST. In total 174 sequences had one or more results. Only the first 30 results per sequence have been saved to the database. If a sequence had less than 30 results, all of its results have been saved. The database can be searched with multiple parameters.

Search: Search the database with multiple parameters. A choice can be made by selecting checkboxes to filter your search in the database, e.g. header, definition etc.
Count: Search the database, and count how many times an item occurs. Only 1 checkbox can be chosen.
Organism: Search the organism database. The choices are: species, genus and family. (See 'what to look for', for more information)
Protein: Search the protein database. The choices are: accession code and definition. (See 'what to look for', for more information)
What to look for: Filter something in the database
If the previous choice was organism: then the options include: a species (e.g. Gammaproteobacteria bacterium), a genus (e.g. Gammaproteobacteria) or a family (e.g. Proteobacteria)
If the previous choice was protein: then the options include either a definition (e.g. outer membrane protein assembly factor BamB) or an accession code (e.g. TDJ28483)
Limit: Limits the amount of results that will be shown in the table.

Search: Search the database with all the selected parameters. The options for what to show in the table:

  • Header: The header of each DNA sequence.
  • Definition: The definition of the protein.
  • Organism_species: The name of the species.
  • Ident_perc: The identity percentage, calculated by the amout of identities divided by the length of the alignment.
  • Accession: The NCBI accession code of the protein.
  • Sequence: The DNA sequence that is used to BLAST.
  • Organism_genus: The name of the genus, which is retrieved from the taxonomie browser.
  • Organism_family: The name of the family, which is retrieved from the taxonomie browser.
  • Query_cov: The query coverage, is calculated by (end of hit minus begin of hit) divided by the length of the DNA sequence.
  • E_value: The e-value which stands for the amount of random sequences in the NCBI database with the same or a higher bitscore.

About

How does it all work and how has it been made?

Morgan

First of all, a Python script was written to automatically BLAST 200 DNA sequences, in which the tool BioPyton was used. All of the data which BLAST provided was saved to an XML file. Then, a Python function was made to parse the XML file, and retrieve the important data. In addition, a Python function was made to automatically insert the retrieved data into a self-created database.
Secondly, this website was created, using Python and the Flask package.

The creators of this product are three individuals known as Yaris van Thiel, Morgan Atmodimedjo and Stijn Bruggink.

The given task was to create a simple interactive method for accessing processed sequencing data through a website. The task was assigned by the HAN Biocentre as a means for them to continue with a assignment from the Productschap Tuinbouw en CNC grondstoffen B.V. The base data received for the project was in a FastQ file and was acquired after paired end illumina sequencing on metagenomic data (paired end illumina sequencing uses two reads for each sequence, a forward and a reverse read with a length of 301 nucleotides).
To get the raw data processed a first requirement was to blast the data using a script and that data had to be saved within a database, which would also require a database. After the database had been created and filled with data the database linkage with the site could be tested before further finishing touches could be applied.
